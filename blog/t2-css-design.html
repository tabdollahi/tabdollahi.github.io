Positioning Techniques in CSS

Difference between Absolute, Relative, and Fixed

Sunday March 8, 2015

CSS offers various techniques for how to position blocks on a webpage. I will be discussing the differences between absolute, relative, and fixed positioning.

Absolute positioning places a page element exactly where you want it. Using the positioning attributes top, left, bottom, and right, you can set the exact location of the element. It will be relative to the next parent element or if the next parent element is the html tag, it will be relative to the page itself. While absolute positioning can be an easy way to place an element exactly where you want it, the downside is that these elements are not affected by other elements and it doesn't effect others. So you can easily get overlaps unintentionally.

Relative positioning allows you to set the positioning of an element relative to where it would normally land on a page. So if you just used position:relative with no other positioning attributes, this would be no different than the default position for a page element. However if you use other positioning attributes, such as left:50px, the element will be positioned 50px to the right of where it would normally be.

Fixed positioning essentially pins an element to a location on the browser window itself. So fixed elements remain in the same place even when the window is scrolled.And there you have it! the difference between absolute, relative, and fixed positioning.
